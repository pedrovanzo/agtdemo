# ADR 0002: JWT Demo Tokens Without a Database

## Status
Accepted

## Context
Access control needed for a public demo without the operational overhead of a database. Tokens must carry expiry and per-user use limits.

## Decision
Self-signed JWTs generated by a localhost-only admin endpoint. Token payload embeds `exp` (expiry) and `max_uses`. Use count tracked in-memory — resets on backend restart, which is acceptable for a demo context.

## Consequences
No database dependency. Simple to operate: boot backend locally, hit `/admin/generate-token`, share token via LinkedIn DM. In-memory counter means use counts are lost on redeploy — acceptable trade-off given demo scope.
